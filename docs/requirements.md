# Requirements (MVP)

Этот документ кратко обобщает требования к MVP проекта **Resume Builder**.  
Полный развёрнутый вариант ТЗ находится в `README.md` (раздел «Техническое задание (MVP)»).

## 1. Цели

- Предоставить веб-интерфейс, где пользователь заполняет резюме.
- Сгенерировать PDF-резюме по LaTeX-шаблону.
- Обеспечить предпросмотр PDF прямо в браузере.
- Поддержать кастомные секции (например, homelab, side projects).
- Фото опционально, автоматически обрезается до 3×4 и сжимается до 2 МБ.

## 2. Границы MVP

- Один шаблон резюме.
- Только английский язык.
- Нет регистрации и личного кабинета.
- Нет постоянного хранения данных (без БД и объектного хранилища).

## 3. Основные сценарии

1. Заполнение формы резюме (личные данные, summary, контакты, навыки, опыт, образование).
2. Добавление кастомных секций с bullet-пунктами и настраиваемым символом bullet.
3. Загрузка фото (опционально).
4. Автоматический предпросмотр PDF (дебаунс).
5. Скачивание готового PDF.

## 4. Нефункциональные требования

- Время генерации PDF: ориентир до 5 секунд.
- Ограничения по длинам полей и количеству записей (см. README.md).
- Фото:
  - любые разумные размеры;
  - если > 2 МБ — сжатие и обрезка до 3×4.
- Экранирование LaTeX-символов.
- Ограничение размера тела запроса (примерно до 256 КБ).

## 5. Стек

- **Фронтенд:** Vue 3 + TypeScript + Vite.
- **Backend API:** Go (REST).
- **LaTeX-сервис:** Go + `pdflatex` + `latexmk`.
- **Reverse proxy:** nginx.
- **Инфраструктура:** Docker, docker-compose, монорепозиторий.

Для деталей по моделям данных и API см. `docs/api.md` и соответствующие разделы `README.md`.
